Getting Set Up — Install Git

Mac Setup
1
Open Terminal Press ⌘ Space to open Spotlight, type Terminal, and hit Enter. This is the command line — don't worry, you'll only need it for this one step.
2
Install Git Type one of these commands and press Enter. Use the first if you don't have Homebrew, or the second if you do. xcode-select --install — or, if you have Homebrew — brew install git
3
Install if needed If git isn't found, macOS will automatically pop up a dialog asking to install the Developer Tools. Click Install and follow the prompts — it takes about 2 minutes. xcode-select --install
4
Confirm it worked Run the check again. You should now see a version number — git is ready to go. git --version

✦  Already have Homebrew installed? You can also run brew install git instead of step 3.

What is Git?

01 / 07
📸

A Time Machine

Git takes snapshots of your project at every save point. You can travel back to any moment — nothing is ever truly lost.

→ Every save = a commit
🌿

A Parallel Universe

You can spin up a separate "branch" to try ideas without touching the stable codebase. Experiment freely, then merge what works.

→ Experiments = branches
☁️

A Shared Drive — With History

GitHub is the cloud home for your git project. Your whole team syncs to it. Unlike Dropbox, it tracks every single change, by whom, and why.

→ GitHub = remote backup + team hub
Git is the tool on your computer.  |  GitHub is the website where you store and share your git project online.

Pull Requests — Proposing Changes

05 / 07
1

Create a Branch

You branch off your team's base branch — usually main, but ask your tech lead which branch to start from. Then work on your feature, fix, or redesign.

git checkout -b feature/hero
2

Make & Push Commits

Do your work, save commits, then push your branch to GitHub.

git push origin feature/hero
3

Open a PR

On GitHub, open a Pull Request. You're saying: "I'd like to merge my branch into the base branch." Your tech lead will confirm where it should land.

GitHub UI → New Pull Request
4

Review & Discuss

Teammates comment, request changes, or approve. Conversations happen inside the PR.

Code review → Approvals
5

Merge!

Once approved, the PR is merged into the base branch. Your work is now part of the shared codebase. Your feature branch can be deleted.

Merge pull request ✓
🔀

A PR is a conversation, not just code.

It's a formal way to say "review my work before it goes live." Always ask your tech lead which branch to target — teams use main, develop, staging, and others. Designers can open PRs for CSS changes, copy edits, and asset updates — not just developers.
