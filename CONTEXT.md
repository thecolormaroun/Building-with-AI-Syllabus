# AI Curriculum

## What This Is
Hands-on workshop series teaching Change.org designers (and PMs) to use Claude Code for prototyping, building, and shipping. Maroun is leading this initiative as part of the broader AI acceleration strategy.

## Curriculum Arc
1. **Session 1** (done, Mar 16): Terminal setup, Claude Code basics, first website from a doc, iteration techniques
2. **Session 2** (done, Mar 31): Git 101, GitHub CLI, commits/branches/push-pull/PRs — taught by Ryan Herubin
3. **Session 3** (up next): Worktrees + Collaborative Coding — hands-on Git practice
4. **Future**: Agentic Coding + Skills, Claude Code ↔ Canvas, Cowork + Agents

## Teaching Philosophy
- Minimal "dangerous" skills — not deep engineering knowledge
- Volume of reps over perfection
- Screenshare-heavy for troubleshooting (10+ participants, all sharing screens)
- Safe-space tone: "I am a designer, I'm unfamiliar with this, explain clearly"
- Three iteration methods: Compound Engineering (/lfg), vanilla Claude Code, Agentation for targeted feedback

## Key Tools
- Claude Code (primary)
- Compound Engineering skill (for full rebuild loops)
- Agentation (for Figma-style feedback on live sites)
- Framer Motion (for adding motion after structure is solid)
- Vercel (deployment target)

## Constraints
- Designers don't have GitHub/Corgi/Vercel access yet — local-only for now
- Don't assume engineering knowledge — explain terminal concepts
- Keep files self-contained (single HTML when possible)
