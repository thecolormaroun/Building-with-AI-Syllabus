What We've Learned So Far

Key commands & techniques.

Setup
Create a home base folder In Finder, go to your user folder (the one with your Mac name). Create a folder called Claude — one word, no spaces. Put it directly in your user folder, not inside Documents or Desktop (Apple has weird permissions on those).
Cmd + Option + C In Finder, copies the full path to any selected file. Use this to point Claude at a document — paste it straight into your prompt.
$ ls Lists what's in your current terminal location. Use it any time you're not sure where you are.
$ cd [foldername] Move into a folder. Example: cd Claude. Everything is case-sensitive — cd claude and cd Claude are different.
$ cd .. Go back up one level.
$ claude Launch Claude Code. Silence = success — the terminal only speaks up when something goes wrong.
3 Ways to Iterate
Voice dump → /lfg Use when the output feels very off and you want a full redo. Takes ~20 min but is thorough. Say: "The nav is completely wrong and the layout feels generic. Redo it using /lfg."
Voice dump → Claude Code Use when it's roughly right but needs a pass. Hold spacebar, describe what's not working, let it take a swing.
/agentation Use for specific element-level feedback. Leave Figma-style comments directly on the live site — Claude knows exactly what you're pointing at.
Say and Show, Don't Tell
Hold spacebar Activates voice input in Claude Code. Talk through what you want instead of typing it — faster, more natural, better output.
Cmd+Ctrl+Shift+4 Screenshot to clipboard. Then paste into Claude Code with Ctrl+V (not Cmd+V — Ctrl+V is for images). Say: "This is what I'm seeing — fix it."
When in doubt, screenshot it Claude can see the image and usually diagnoses broken layouts or errors immediately. Almost always faster than describing the problem in words.
Slash Commands
/lfg Full autonomous build: plan → build → review → ship. Takes ~20 min. Use it when starting fresh or doing a full redo.
/review Multi-agent code review — catches bugs, security issues, and code quality problems.
/clear Reset the conversation context. Use when Claude starts going in circles or forgetting earlier instructions.
Git & GitHub (Session 2)
$ xcode-select --install Install Apple's command line tools (includes Git). One-time setup — if it says "already installed," you're good.
$ brew install gh Install the GitHub CLI. Lets you interact with GitHub from the terminal.
$ gh auth login Authenticate with GitHub. Pick GitHub.com → HTTPS → Login via browser. Enter the one-time code it gives you.
$ git --version Check if Git is installed. You should see a version number.
$ gh api user Verify your GitHub connection. Shows your username and profile info.
Commit A snapshot of your code at a point in time — like a save point in a video game. Claude Code creates these for you.
Branch A parallel copy of the codebase. Make changes without affecting the original. Branch off main by default.
Push / Pull Push = upload your changes to GitHub. Pull = download latest changes from GitHub. Keeps local and remote in sync.
Pull Request (PR) A formal request to merge your branch into main. Opens a conversation with engineering — they review, approve, and merge.
Remote / Origin Both mean GitHub (for now). It's where your code lives in the cloud.
Claude Code handles Git You don't need to memorize commands. Tell Claude: "Create a branch", "Commit my changes", "Am I behind the remote?" — it speaks Git natively.
