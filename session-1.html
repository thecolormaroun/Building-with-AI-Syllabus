Self-Guided Walkthrough

Do it yourself, step by step.

Everything we covered in session 1, written out step by step. For anyone who missed it — or wants to run through it again at their own pace.

Step 1 Create your home base folder

Open Finder. Navigate to your user folder — this is the folder named after your Mac username (the one that contains Applications, Desktop, Documents, etc.). You can get there by right-clicking the folder name at the top of any Finder window and selecting your username from the dropdown.

Inside that folder, create a new folder. Name it Claude (or Mission Control, or Brain — whatever works for you). One word only, no spaces — it'll be easier to reference later.

This folder is where everything lives. All your Claude projects, all your memory, everything you build.

⚠️ Watch out: Don't put this folder inside Documents or Desktop. Apple has weird permissions on those folders that can cause issues. Put it directly inside your user folder.
Step 2 Open Terminal and navigate to your folder

Search for Terminal in Spotlight (Cmd+Space) and open it.

The terminal is always "located" somewhere in your file system. You need to move it into your new Claude folder. There are only three commands you need to know:

  • ls — lists what's in your current location. Use this any time you're not sure where you are.
  • cd [foldername] — moves you into a folder. Example: cd Claude
  • cd .. — moves you back up one level

Type ls and press Enter. You should see your user folders listed. If you see Claude in the list, type cd Claude and press Enter.

⚠️ Watch out: Everything in terminal is case-sensitive. cd claude and cd Claude are different things. If it says "no such file or directory," check your capitalization.
⚠️ Watch out: Terminal gives almost no confirmation when things work. Silence = success. You'll only see output when something goes wrong.
Step 3 Install Claude Code

Go to the #pd-ai Slack channel and find the pinned setup message. Copy the first install command (it starts with curl).

Paste it into your terminal with Cmd+V and press Enter. This will download and install Claude Code. You'll see a bunch of text scroll by — that's normal. Wait for it to finish.

When it's done, you'll see a line that says something like echo export PATH=.... Copy that entire line and paste it back into the terminal, then press Enter.

⚠️ Watch out: That echo line must be on a single line. If your terminal window is narrow, it might appear to wrap — make the window wider and try again if you get an error.
⚠️ Watch out: If it asks you to install Homebrew as part of this process, say yes and let it run. Homebrew is like an app store for development tools. Claude Code will need it.
Step 4 Summon Claude Code

Type claude (all lowercase) and press Enter.

A setup wizard will appear. Use your arrow keys to navigate and Enter to select. First choice: light or dark mode. Second: it will ask you to log in. Select the option to log in via browser, click Authorize, and come back to the terminal.

⚠️ Watch out: Make sure you log in with your Change.org Google account, not a personal account. If you accidentally log in with a personal account, you'll be on the free plan and some features won't work. Exit out (Ctrl+C), reopen terminal, cd Claude, type claude again.

When you're in, it will ask about your workspace. Select "Trust this folder" — this tells Claude Code it's allowed to read and write files within your Claude folder.

You're in. Claude Code is now running like a chatbot in your terminal.

Step 5 Install your skills

Skills are like plugins that give Claude Code superpowers. Today we install two.

For each skill: Go to the Slack channel and find the GitHub link for the skill. In Claude Code, type something like "I want you to install this skill so we can use it going forward" and paste the GitHub URL. Claude will read the code, confirm it's safe, and install it.

Skill 1: Visual Explainer — Turns any document into an interactive, visual website. It has modes for slides, diagrams, and full-page layouts.

Skill 2: Compound Engineering — Your workhorse. Installs a full multi-agent development workflow. The key command is lfg — when you put that at the start of a prompt, it runs through a full plan → build → review loop without stopping to ask questions. It takes ~20 minutes but produces thorough, well-structured results.

⚠️ Watch out: These skills are large and take a few minutes to install (Compound Engineering especially — up to 4-5 minutes). Let them run. Don't interrupt.
⚠️ Watch out: After installing each skill, you need to restart Claude Code. Close the terminal window, reopen it, cd Claude, type claude.
Step 6 Install Agentation (feedback tool)

Agentation is a third skill that lets you leave Figma-style comments directly on your live website in the browser and pipe that feedback back to Claude Code. Install it the same way as above using the link in the Slack channel.

To activate it, type /agentation in Claude Code. It will open a special browser window with your site loaded. A small toolbar will appear — click on any element to leave a comment. When you're done commenting, hit "Copy Feedback" in the toolbar, paste it into Claude Code, and it'll address all of your notes at once.

⚠️ Watch out: If the site loads but all the elements appear bunched in the bottom-left corner, screenshot the whole page, paste it into Claude Code (Ctrl+V for images), and say "for some reason all the elements are rendering in the bottom left corner, can you fix this?"
Step 7 Find your content and build your first site

Find something to turn into a website. Good options:

  • → A strategy doc (download as Markdown from Google Docs — File → Download → Markdown)
  • → A research brief or research findings doc
  • → A prioritization spreadsheet (download as CSV)
⚠️ Watch out: If you try to download a Google Doc as Markdown and it comes out as a 0-byte empty file, use PDF instead — it works too.

Point Claude to your file: In Finder, click on your file to highlight it. Press Cmd+Option+C. This copies the file's location (its path) as text. Now paste it (Cmd+V) into your Claude Code prompt.

Your first prompt: Hold the spacebar to activate voice input. Say something like:

"I want you to build a website from this document. Make it interactive and use the Visual Explainer skill. Use as much layout as possible — don't just give me a wall of text. If you see an opportunity to build a diagram or make something interactive, take it. Use the words from the document verbatim. Here's the file:" → then paste the path.

Press Enter and let it cook. It'll take a few minutes.

⚠️ Watch out: When prompts come up asking for permissions (like "can I read this file?"), always choose option 2: Yes, and don't ask again for this session. This keeps the workflow moving.
Step 8 Preview your site

When Claude is done, it will have created an HTML file inside your Claude folder. Find it in Finder and double-click it — it will open in your browser.

Alternatively, Claude Code may tell you it's running at http://localhost:3000 or similar. Copy that URL and paste it into your browser.

You now have a live website running locally on your machine. Nobody else can see it yet — that's fine for now.

Step 9 Iterate on it — 3 methods

Now the real work begins. Here are the three ways to give feedback, and when to use each:

Option 1 — Voice dump → Compound Engineering (/lfg)

Use when: the output feels very off and you want a full redo. Takes ~20 min but is thorough. Say: "This isn't working for me. I want you to redo this using the LFG command from Compound Engineering."

Option 2 — Voice dump → vanilla Claude Code

Use when: the output is roughly right but needs improvement and you don't want to wait 20 minutes. Just hold spacebar, describe what you want changed, and let it take a pass.

Option 3 — Agentation → targeted feedback

Use when: you have specific feedback on specific elements. Type /agentation, leave comments in the browser toolbar, hit Copy Feedback, paste into Claude Code.

Step 10 Add motion

Once your site looks good, ask Claude Code to add motion:

"I want you to install Framer Motion and add tasteful hover effects, scroll animations, and transitions to this site."

It will install Framer Motion (via npm — like Homebrew but for JavaScript packages) and wire it into your site.

Then creative edit the motion. Tell it to make things faster, slower, more subtle, or remove effects that feel cheap. This is where your designer taste really matters — Claude will do whatever you ask here.

⚠️ Watch out: The first time Claude needs Homebrew or npm, it'll ask you to install them. Just say yes and let it handle it. Once they're installed, you'll never need to think about them again.
Golden Rule How to troubleshoot when things go wrong

Any time something looks broken, weird, or you're stuck — take a screenshot (Cmd+Ctrl+Shift+4 to copy to clipboard) and paste it into Claude Code with Ctrl+V (not Cmd+V — Ctrl+V is for images). Say: "This is what I'm seeing — can you tell me what's going on and fix it?"

Claude can see the screenshot and usually diagnoses the problem immediately. This is almost always faster than trying to describe it in words.

You can also type btw in Claude Code while it's working on something to have a side conversation without interrupting the main task.

Use for general feedback where you don't need the full pipeline. Just hold spacebar, describe what's not working, and let it take a pass.

Pro tip

Screenshot everything. Whenever something looks broken, weird, or you're not sure what Claude or the terminal is doing — screenshot it and paste it straight into Claude Code (Control+V for images). Ask it what's going on or tell it to fix it. This is almost always faster than trying to describe the problem in words.

A note on tooling

As you keep building, Claude Code will likely need to install Homebrew and npm — think of these as app stores for development tools. Don't stress when they come up. Just ask Claude Code to walk you through it.

It can also help to tell Claude: "I am a designer, I'm unfamiliar with this stuff — explain things clearly, try to fix things yourself, and give me terminal commands one at a time. Add this to your Claude.md so you remember."
